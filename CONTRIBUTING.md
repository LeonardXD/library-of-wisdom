Contributing to The Shared Library of Wisdom

Thank you for contributing to The Shared Library of Wisdom!
This document outlines the rules and guidelines that all contributors must follow to ensure smooth collaboration and proper version control.

1. Branching Rules

Direct commits to the main branch are strictly prohibited.

All work must be done in a feature branch.

Branch naming format:

feature/<your-name>

2. Making Changes

Keep changes focused on your assigned task only.

Do not modify files assigned to other members unless instructed.

Ensure files follow proper Markdown formatting where applicable.

3. Commit Message Guidelines

Commit messages must be clear and descriptive.

Use short, meaningful messages.

Good examples:

Add contributing guidelines

Create history.md with Git facts

Bad examples:

update

fix

changes

4. Pull Request (PR) Rules

All changes must be submitted via a Pull Request.

The PR description must include:

A brief summary of what was changed

The purpose of the change

Each Pull Request must receive at least one review comment before merging.

5. Code Review Etiquette

Review teammates’ Pull Requests respectfully and constructively.

Leave at least one meaningful comment before approving.

Do not merge your own Pull Request without review.

6. Merge Conflict Handling

If a merge conflict occurs:

Pull the latest main branch into your feature branch.

Resolve conflicts manually in your editor.

Ensure no teammate’s work is deleted.

Commit and push the resolved changes.

Ask for help if you are unsure how to resolve a conflict.

7. General Conduct

Follow collaboration best practices.

Communicate clearly with your team.

Respect deadlines and assigned responsibilities.
